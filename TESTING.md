# Тестирование приложения

В этом проекте используются два типа тестов: unit-тесты и интеграционные тесты. Они различаются по цели, области тестирования и используемым инструментам.

## Unit-тесты

Unit-тесты тестируют отдельные компоненты приложения (функции, классы, методы) изолированно от остальной системы. Они запускаются в окружении JSDOM (виртуальный DOM в Node.js) и не требуют запуска реального браузера.

### Особенности:
- Запускаются с помощью JSDOM (виртуальный DOM в Node.js)
- Быстрые, так как не требуют запуска браузера
- Тестируют изолированно от браузера и других систем
- Используют mock-объекты для имитации взаимодействия с DOM
- Именуются по шаблону: `*.test.js` (кроме файлов с `puppeteer` в названии)

### Примеры:
- `ProductModel.test.js` - тестирует классы Product и ProductList
- `View.test.js` - тестирует класс View с mock-объектами DOM

### Запуск:
```bash
# Запуск всех unit-тестов
npm run test:unit

# Запуск unit-тестов с отслеживанием изменений
npm run test:unit:watch

# Запуск unit-тестов с покрытием кода
npm run test:unit:coverage
```

## Интеграционные тесты

Интеграционные тесты проверяют взаимодействие между различными компонентами приложения в реальных условиях. Они запускаются в реальном браузере с помощью Puppeteer и тестируют приложение как единое целое.

### Особенности:
- Запускаются в реальном браузере (Chrome) через Puppeteer
- Медленнее unit-тестов, так как требуют запуска браузера
- Тестируют приложение в реальных условиях
- Проверяют взаимодействие с DOM, обработчики событий и пользовательские сценарии
- Именуются по шаблону: `*.puppeteer.test.js`

### Примеры:
- `integration.puppeteer.test.js` - тестирует полный пользовательский сценарий работы с приложением

### Запуск:
```bash
# Запуск всех интеграционных тестов
npm run test:integration

# Запуск интеграционных тестов с отслеживанием изменений
npm run test:integration:watch

# Запуск интеграционных тестов с покрытием кода
npm run test:integration:coverage
```

## Когда использовать каждый тип тестов

### Unit-тесты:
- При тестировании бизнес-логики
- При тестировании отдельных функций и методов
- Для быстрой проверки корректности работы отдельных компонентов
- При необходимости быстрого запуска тестов во время разработки

### Интеграционные тесты:
- При проверке пользовательских сценариев
- При тестировании взаимодействия между компонентами
- При проверке работы с DOM и обработчиков событий
- При тестировании сложных сценариев, требующих реального браузера
